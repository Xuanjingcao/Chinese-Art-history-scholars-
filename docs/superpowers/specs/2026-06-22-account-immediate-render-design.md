# 账户页立即显示设计

## 背景与根因

进入“我的账户”时存在两层可见等待：`App` 懒加载账户页面时显示“正在打开账户...”，账户组件挂载后又用“加载中...”替代整个页面。账户组件还先等待用户资料，再并行请求收藏、浏览记录、投稿、笔记、已发布帖子和草稿；最慢的一项返回前，页面主体始终不可见。

## 方案比较

1. **立即渲染、会话缓存、后台刷新（采用）**：账户框架立即出现；优先展示同一会话中的缓存，再在后台更新。首次访问没有缓存时也不遮挡页面，各区域先显示现有空状态。
2. **保留整页等待，仅改成骨架屏**：视觉更柔和，但仍然要等待最慢请求，不能解决用户无法立即进入账户页的问题。
3. **只并行化请求**：能缩短部分耗时，但 CloudBase 网络稍慢时仍会显示整页“加载中”，不能保证目标体验。

## 设计

### 页面加载

`MyAccountPage` 改为普通静态导入，移除账户分支的 `Suspense`。账户页面组件挂载后立即渲染标题、返回按钮、资料区和各内容区，不再存在整页加载分支。

### 数据流

为账户页增加按用户隔离、带版本号的 `sessionStorage` 快照。组件通过惰性初始状态同步读取快照；有缓存时先显示缓存，没有缓存时使用空数组和现有空状态。随后同时启动用户资料及六组账户数据请求，避免先查询用户资料造成瀑布等待。

后台请求成功后一次性更新对应状态并写入会话缓存。页面停留期间的删除收藏、清空历史和编辑资料操作也同步更新缓存，避免返回账户页时短暂恢复旧数据。

### 错误处理

某一类数据请求失败时，不让其他成功请求一并失败。失败区域保留缓存或当前内容，成功区域照常刷新；错误记录到控制台用于排查，但不把整个账户页清空，也不重新显示整页加载提示。存储不可用或缓存损坏时自动忽略缓存，继续正常请求。

### 范围约束

本次只处理账户页的可见等待和请求瀑布，不改动 CloudBase 数据结构、登录规则或其他页面的加载提示。账户页面仍然仅对已登录用户开放。

## 验收

- 点击“我的账户”后立即显示账户页，不出现“正在打开账户...”或整页“加载中...”。
- 用户资料和六组数据请求并行启动，单项失败不会清空其他内容。
- 同一标签页再次进入账户页时先展示缓存内容，再后台更新。
- 自动测试覆盖账户页非懒加载、无整页加载分支、缓存隔离和损坏回退。
- 完整运行 `npm test`、`npm run build`，并在本地浏览器验证账户跳转及控制台状态。
